⚖️ The Grammar Workshop — Lexon x the Privacy Value Model

What if the ideas behind a privacy architecture could be written in a language that is at once plain English, an executable contract, and a machine-checkable claim? This workshop expressed the Privacy Value Model (the theory behind 0xagentprivacy: two separated agents, a Swordsman who protects and a Mage who delegates, held apart by a Gap) term by term in Lexon, Henning Diedrich's controlled-English language for computational law.

The result of six days: 189 of the model's 211 canon terms are now expressed in a form whose load-bearing claim can be mechanically verified and falsified. The census is 90 percent covered. The harness even wrote its own constitution in the language its own loop gates.

Source of truth: github.com/mitchuski/lexon_pvm (git). This wiki is a forkable projection that carries lineage.

One paragraph, if you read nothing else

The Privacy Value Model is a theory of privacy as architecture: two software agents kept structurally separate so neither can learn what the other knows. This workshop expressed that model's canon in Lexon, a language that reads as plain English but parses to knowledge-graph triples, using a research loop built on the same two-agent design. Every expression carries a machine-checkable claim, so the model's promise that a boundary "cannot be crossed by design" becomes a query that provably returns nothing. What remains uncovered is the mathematical frontier and the deep story, left home on purpose.
