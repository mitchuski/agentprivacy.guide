Quaternion Mapping

Category role · Tier 1 · Version 5.3.2

Equation term Quaternion symmetry in dual-agent derivation

Domain skill for four-body system analysis and mapping. Covers the Sun-Earth-Moon-Human quaternion, generator/generated relationships, and how two conflicting constraints produce two derived agents through different creation modes (impact vs process).

☀️🌍🌑👤 Quaternion Mapping — Four Bodies, Two Dualities

"When two conflicting constraints find balance, they create from their orbit."

Core Principle

The quaternion is a complete symmetry group for privacy architecture:

  • Two generators (Sun, Earth) — the original duality
  • Two generated (Moon, Human) — the derived agents
  • One gap (⊥) — the irreducible separation

This skill enables mapping any dual-agent system to the quaternion structure and deriving architectural constraints from the mapping.

The generators are in orbit—perpetual tension, neither consuming the other. Their balance creates the conditions for derivation.

  • Moon inherits Sun's function (reflection) but serves Earth
  • Human inherits Earth's function (connection) but looks toward Sun

Mapping Rules

Generator Identification

To map a system to quaternion structure, first identify generators:

  1. Protection constraint — What burns/consumes to enable others?
  2. Delegation constraint — What lives/generates to produce complexity?

These must be in stable orbit—neither can consume the other without destroying the system.

Derivation Mode Analysis

Once generators identified, analyze how agents derive:

Mode Speed Amnesia Example
Collision Instant Complete Key ceremony, credential issuance
Process Extended Gradual Training, comprehension, trust-building

Systems need both modes:

  • Collision-derived agents for immediate separation (Swordsman)
  • Process-derived agents for gradual capability (Seeker → Blade-holder)

Gap Persistence

The gap between generated agents is permanent because:

  • Collision-derived cannot remember (Moon)
  • Process-derived hasn't finished (Human)

Attempts to close the gap produce merge catastrophe.

Operational Patterns

When to Apply Quaternion Mapping

  1. System Design Validation

    • Map proposed architecture to quaternion
    • Verify both generators present and in tension
    • Confirm both derivation modes available

  2. Role Assignment

    • Determine which components are generators vs generated
    • Assign protection/delegation functions correctly
    • Ensure gap maintenance in generated agents

  3. Trust Analysis

    • Collision-derived trust: structural, immediate
    • Process-derived trust: earned, gradual
    • Both required for complete system

Quaternion Completeness Check

A privacy system is quaternion-complete if:

  • Two generators in orbit — Protection and delegation in tension
  • Collision derivation path — For immediate agent creation
  • Process derivation path — For gradual capability building
  • Gap maintained — Generated agents cannot merge
  • Function inheritance correct — Derived serve opposite generator
