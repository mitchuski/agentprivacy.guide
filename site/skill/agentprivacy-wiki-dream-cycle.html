Wiki Dream Cycle

Category wikis · Version 5.5 · Keeper the Librarian 🗃️ · Author the Chronicler 🧙📖

The standing research loop that keeps the static guide rebound to the source. Each relevant /mitch repo dreams about itself — a deep multi-lap pass that diffs the repo’s canon against the live FedWiki, writes a standing chronicles/DREAM-<date>.md (its resumable work-ledger), and feeds a single confirm-able GATE document. The First Person ticks the GATE; builders write the approved canon into ~/.wiki; the snapshot publishes it.

Where Wiki Resync is the per-turn operation (“I changed a repo → rebuild the wiki”), the dream cycle is the discovery layer that runs before you know what to change. It is the Chronicler’s loop — it produces chronicles — cataloged by the Librarian, keeper of the wikis category.

The four stages (looped)

1. RESEARCH   scan source dirs → diff vs the live FedWiki → gap report
2. GATE       the First Person reads the dreams and ticks what to fill  ← the only human step
3. BUILD      builders write the approved canon into ~/.wiki (journal-versioned, <a class="wlink" href="linked.html">linked</a>, hub backlinks)
4. SNAPSHOT   snapshot → site → review on :3200
   ↺ loop until comfortable → push → guide.agentprivacy.ai

Two layers of dream

  • Thematic dreams — cross-cutting passes by site/topic, plus a relevance-curated wider-universe pass that triages the ~80 /mitch dirs into current · stale · irrelevant so the fan-out never dreams on cruft.
  • Per-directory dreams — each current repo dreams about its own state + its link to the guide, into <repo>/chronicles/DREAM-<date>.md. The daily work-ledger.

Discipline

  • Relevance first — never treat a stale/version-named dir or system folder as a gap; triage, then verify against the live sites.
  • No duplicate information flows — fill what is missing; link, don’t re-create, what already lives elsewhere.
  • The dreams are research input, not content to paste.
  • The human gate is load-bearing — nothing built or published without the First Person.
  • Honest framing carries — every dream preserves the source’s honesty.

Run it

node flow/run.mjs audit          # RESEARCH → gap-report.json
node flow/run.mjs build <site>   # BUILD the approved gaps into ~/.wiki
node flow/run.mjs snapshot       # SNAPSHOT → site → review on :3200
node flow/run.mjs cycle <site>   # audit → build → snapshot → audit
