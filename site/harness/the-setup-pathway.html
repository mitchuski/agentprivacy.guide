🚪 The Setup Pathway — clone to your own harness

What this is. A verification harness for AI-agent work: one agent proposes, a second independently proves, and the tests that decide are derived by hashing the proposal itself — so neither agent could have chosen them. Config-driven, zero dependencies, every axiom checked at runtime. Full statement: The Dual-Agent Harness.

Where it stands. Debugged by being run — seven defects found on execution, each fixed and pinned. The toy has a real advancing frontier (see The Field Guide Spar, numbers cited live). Ten embodiments catalogued in The Harness Paths. The skeleton is domain-neutral and stays that way.

Requirements. Node ≥ 18, nothing else. Multi-agent rounds need a seat driver — Claude Code's Workflow tool is the reference runtime; the engine also accepts any rt = { agent, parallel, pipeline, phase, log } you supply.

The seven steps

  1. Prove the axioms — node tools/check.mjs: the ℤ/64ℤ algebra, the engine's failure-semantics tests, conformance of every instance, and the negative test that the blank template fails for the right reasons. See The Engine And Its Gates.
  2. Read the constitution — The Six Trusts, then The Ten Ground Rules. These are the parts you should not change.
  3. Run the toy — a real round against the field guide: Workflow tool, scriptPath: examples/field-guide/harness.workflow.mjs, args: {repo, root, runId}. All writes land in runs/<runId>/; the frontier does not move on its own. See The Field Guide Spar.
  4. Audit and view — node tools/render_run.mjs examples/field-guide <runId> re-derives every Gap seed from the saved proposal bytes; or sha256sum proposal_canon.json = seedHex by hand.
  5. Fold as keystone — conform green → fold the validated-and-structural lever → frontier first, prose second → chronicle → conform green again. See Seat — Keystone and The Ledgers. Publishing anything is the First Person's alone (T6); where the claim space is enumerable, run the census first (what you can count, count).
  6. Seal what survived — mint_artefact.mjs writes a κ-labelled artefact bundle (content-addressed, never trusted only re-derived) with an evidence hash-manifest and a DOOR.md of the actions software did NOT take; it refuses tampered seeds, red gates, and verdict-less runs. Bundles surface in The Workshop Console.
  7. Scaffold your own — node tools/new_instance.mjs ../my-harness my-harness, fill every TODO (gate and bundler both refuse a config still wearing them), bundle, run like the toy. The Seat Contract is the interface and Adopting The Harness is the map — the harness-vs-auditor decision, the five answers that become your config, and six worked domain mappings. Define the Gap first — no hash-derived witnesses, no harness; you may want an auditor instead (Harness Path — The Universe Builder Retired).
  8. Specialise when it earns it — Specialisation — The Game Of 42 and Mage And Swordsman; The Harness Paths shows ten real configs at every weight.

Follow the story

The pathway has been walked end to end under its own gates: the spar's frontier records four audited folds — 730 → 573 → 526 → 472 words, every step at gate 8/8 on hash-drawn witnesses, the last two closed by exhaustive census — numbers cited live on The Field Guide Spar. The loop found seven of its own defects by running; one round was watched live in The Workshop Console; two κ-labelled artefacts sit PREPARED at the door; one seat is held open by invitation (Harness Path — Hearthold The Open Slot). The chronicles in the repo are the record, read them in date order — the wiki is the map.

The whole pathway in one sentence

Prove the axioms, read the trusts, run a round you can audit by hash, fold only what two separated agents agreed on, and only then make it yours — your harness path is a config, not a fork.
