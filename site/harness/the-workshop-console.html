🖥️ The Workshop Console

node tools/console.mjs → http://127.0.0.1:4242 — a localhost window onto any harness instance. A window, not a hand: the server is GET-only by construction (there is no route that writes, folds, mints, or publishes), and it holds no state — every poll re-reads the run directories and re-derives every Gap seed with node:crypto, rendering MATCH in cyan and MISMATCH as a coral ROUND VOID (Law L5: never trusted, only re-derived).

The three instruments

  • The Loop — the six phase stations as a ring with the keystone at center and the door drawn as a literal gap in the ring. Stations light purely from file appearance: violet sparks per proposal, chords to the Gap (solid cyan on MATCH, broken coral on MISMATCH), verdict colors per assay, the chronicle station on its draft. The keystone glows only when computed — frontier best ∩ the round's levers.
  • The Frontier — the moving ceiling: baseline hairline, the best-line stepping down through closed targets, the open target as a violet dashed ceiling, and MIRAGE ticks sitting in amber below it — the multiplicative gate made visible (T5: a zero collapses the product).
  • The Lattice — only when the universe/ seam is present: ℤ/64ℤ as seven popcount rings, axis glyphs read from universe.json (never hardcoded — the E-7 lesson), seated XOR-63 pairs as literal diameters, vacancies as dashed vertices — including the hearthold slot held for the House of Archon — and the axiom coin stepping succ while bnot∘neg traces the same step.

Instructional monitoring

Click any station and its seat card opens beside the geometry; the six trusts render as chips that claim only what was mechanically checked (T2/T5/T6 from the config and the conformance gate; T3/T4 as honest grey "topological / documented"). A "run every gate" button prints tools/check.mjs verbatim — the console never summarizes a gate it didn't run. Point it at any instance anywhere: node tools/console.mjs --root <dir> discovers by the same rule as the gate (a directory carrying harness.config.mjs).

Artefacts and the door

node tools/mint_artefact.mjs <instance> <runId> seals a run into artefacts/<id>/ — a κ-labelled record (sha256 over canonical JSON, the label excluded from its own preimage — the same content-addressing as the City Key), byte copies of all evidence with a hash manifest, and a DOOR.md listing the outward actions the software did NOT take. It refuses a tampered seed (GR-4), a red conformance gate (G0), or a verdict-less run (GR-5). When the universe layer is present the artefact carries drafts that track into the universe — a spellweb node draft, a FedWiki page draft, a Gatehouse-eligible flag, a seated-vertex hint — and the edge rule travels with them: a validated round PROPOSES an edge; only a signature MINTS it. The console's door panel re-derives every κ on display and shows the copy-paste commands. Nothing on the page executes any of them.
