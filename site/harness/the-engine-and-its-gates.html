⚙️ The Engine And Its Gates

Zero-dependency Node, three files, every claim checked rather than trusted. This page is the map; the code is the source of truth — read it in the repo.

engine/dual_agent_loop.mjs — the loop

The generic six-phase engine: takes a harness.config.mjs (see The Seat Contract) plus {repo, root, runId} and runs Measure → Propose(∥) → Hold-apart → Assay → Critic → Chronicle. Propose→Assay is a pipeline with no barrier. It returns verdicts and drafts; it never folds, never writes ledgers. A round that loses a seat returns status: INCOMPLETE with keystoneTodo: FOLD NOTHING — empty is evidence; dead is not.

engine/conform.mjs — the conformance gate

Proves the axioms hold before any work: the algebra neg(bnot(x)) = succ(x) on all of ℤ/64ℤ, the six trusts anchored, GR-1..10 present, seven seats carded, door: 'first-person' literal, instance config + frontier coherent. It keeps its OWN copy of the axioms and deliberately does not import them from the engine — both are checked against a third thing. It refuses a config still wearing its template TODOs: a gate that passes an unfilled config is worth less than no gate, because it is believed.

engine/loop.test.mjs — the failure semantics, pinned

Six tests that exist because of real failures: an outage must never be reported as an exhausted search; a dead seat is not a dry round; the blank template must fail conform for the right reasons (asserted by mutation, not by hope).

tools/check.mjs — one command, every gate

node tools/check.mjs — proves the algebra, runs the engine tests, conforms every instance it discovers, asserts the template fails, audits universe/ if present. If it exits non-zero, every line it printed is a command you can re-run yourself.
