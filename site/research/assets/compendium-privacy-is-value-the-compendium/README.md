# Privacy is Value: The Compendium

**What this is:** the assembly point for the one publishable body of work. The compendium does not copy the corpus; it ASSEMBLES it. Each Part carries a `PART.md` manifest listing its sources in reading order; the build script (phase 3) concatenates from the manifests; new prose exists only where the book needs connective tissue (front matter, era retrospectives, back matter). The corpus stays living; the book is a reading of it.

## Rules

1. Manifests list paths into the living corpus; the book inherits edits automatically at next build.
2. A source enters a manifest only with its class tag (S · R · A · M · N) per the constitution.
3. Era-bound material (Class R, M) freezes with its era; spine material (Class A) is always current.
4. Nothing in `compendium/` restates model content; restatement is what the corpus is for.
